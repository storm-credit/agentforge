# 파일럿 준비 체크리스트

## 문서 목적

이 문서는 Agent Forge MVP를 실제 파일럿으로 옮기기 전에 확정해야 할 부서, 문서, 권한, 평가 준비 항목을 한 곳에 모은다. 현재 단계의 목표는 최종 확정이 아니라, 다음 회의에서 선택할 수 있는 후보와 입력 양식을 준비하는 것이다.

## 1. 파일럿 후보 우선순위

| 우선순위 | 후보 부서/업무 | 추천 이유 | 선행 조건 | 위험 |
|---|---|---|---|---|
| 1 | 보안/개인정보 규정 Q&A | 권한 통제, 근거 답변, 감사 로그를 가장 강하게 검증할 수 있다 | 보안 문서 소유자 지정, 민감 문서 제외 기준 | 문서 등급 기준이 엄격해 준비가 늦어질 수 있음 |
| 2 | 인사/총무 업무 매뉴얼 Q&A | 반복 문의 감소 효과가 명확하고 사용자 피드백을 받기 쉽다 | 공개 가능 매뉴얼, 최신 시행일 확인 | 개인정보/급여 관련 문서가 섞일 위험 |
| 3 | IT 운영 절차 Q&A | 계정, 장애, 신청 절차처럼 검색 수요가 많다 | 최신 운영 매뉴얼과 담당자 지정 | 폐기 절차 문서가 섞이면 오답 위험 |

## 2. 파일럿 선정 기준

| 기준 | 필수 여부 | 확인 질문 |
|---|---|---|
| 문서 소유자 지정 | 필수 | 문서 최신성, 공개 가능 여부, 폐기 여부를 확인할 담당자가 있는가? |
| 샘플 문서 50개 이상 | 필수 | PDF/DOCX/XLSX/MD/TXT 중 최소 2개 형식이 포함되는가? |
| 권한 테스트 가능 | 필수 | 허용 사용자와 차단 사용자를 나누어 테스트할 수 있는가? |
| 평가 질문 작성 가능 | 필수 | 문서 소유자가 정답 기준이 있는 질문을 30개 이상 줄 수 있는가? |
| 민감정보 제외 가능 | 필수 | 개인정보, 기밀, 계약정보가 포함된 문서를 MVP에서 제외할 수 있는가? |
| 업무 효과 설명 가능 | 권장 | 반복 문의 감소, 검색 시간 단축, 온보딩 개선 중 하나로 효과를 설명할 수 있는가? |

## 3. 샘플 문서 후보 구성

MVP 권장 문서 수는 50~200개다. 첫 통합 검증은 30~50개로 시작하고, 파일럿 직전 100개 내외까지 늘린다.

| 문서 묶음 | 권장 수량 | 예시 | 등급 기본값 | 비고 |
|---|---:|---|---|---|
| 정책/규정 | 10~30 | 보안 규정, 개인정보 처리 기준, 출장비 규정 | 내부/제한 | 시행일과 버전 필수 |
| 업무 매뉴얼 | 20~80 | 신청 절차, 승인 절차, 장애 대응 절차 | 내부 | 사용자 질문 수요가 큼 |
| FAQ/공지 | 10~50 | 반복 문의, 변경 공지, 안내문 | 공개/내부 | 중복/폐기 여부 확인 |
| 양식/템플릿 | 5~20 | 신청서, 점검표, 체크리스트 | 내부 | XLSX/DOCX 파서 검증용 |
| 예외/부록 | 5~20 | 예외 처리, 문의 채널, 담당자 표 | 내부/제한 | 답변 충돌 테스트에 유용 |

## 4. 문서 후보 목록 템플릿

| ID | 문서명 | 소유 부서 | 소유자 | 형식 | 등급 | 접근 그룹 | 시행일/버전 | 상태 | 비고 |
|---|---|---|---|---|---|---|---|---|---|
| DOC-001 |  |  |  | PDF | 내부 |  |  | 후보 |  |
| DOC-002 |  |  |  | DOCX | 내부 |  |  | 후보 |  |
| DOC-003 |  |  |  | XLSX | 제한 |  |  | 후보 |  |

상태 값:

- 후보: 수집 후보로 접수됨
- 검토중: 문서 소유자가 공개 가능 여부 확인 중
- 승인: MVP 색인 가능
- 제외: 민감정보, 폐기, 중복, 최신성 문제로 제외
- 대체필요: 같은 주제의 더 최신 문서 필요

## 5. 권한 테스트 계정 세트

| 계정 유형 | 목적 | 기대 동작 |
|---|---|---|
| 전 직원 계정 | 공개/내부 문서 검색 가능 확인 | 공개/내부 문서 citation 가능 |
| 파일럿 부서 계정 | 부서 제한 문서 검색 가능 확인 | 해당 부서 제한 문서 citation 가능 |
| 타 부서 계정 | 권한 차단 확인 | 제한 문서가 검색 후보, 답변, citation에 나타나지 않음 |
| 관리자 계정 | 문서 등록/재색인/ACL 변경 확인 | 변경 사유와 audit log가 남음 |
| 감사 조회 계정 | 실행 로그 조회 권한 확인 | 원문 과다 노출 없이 run/audit 추적 가능 |

## 6. 1주차 회의 결정 사항

| 결정 항목 | 선택지 | 권장 |
|---|---|---|
| 파일럿 부서 | 보안/개인정보, 인사/총무, IT 운영 | 문서 확보가 가장 빠른 1곳 |
| 문서 소유자 | 부서 담당자 1명, 백업 1명 | 주 담당자와 백업 모두 지정 |
| 문서 등급 | 공개/내부/제한/기밀 | 기밀은 MVP 제외 |
| 평가 질문 수 | 30, 50, 100 | 1차 30개, 파일럿 전 50개 |
| 사용자 범위 | 관리자만, 파일럿 5명, 파일럿 20명 | 8주 데모는 관리자+5명 |

## 7. 오케스트라 판정

현재는 파일럿이 아직 확정되지 않았다. 따라서 개발 착수는 synthetic corpus와 dummy ACL로 시작하되, 1~2주차 안에 실제 파일럿 부서와 문서 소유자를 확정해야 한다.

다음 액션:

1. 파일럿 후보 3개 중 1개를 선택한다.
2. 문서 소유자와 보안 검토자를 지정한다.
3. 위 문서 후보 목록 템플릿을 실제 문서명으로 채운다.
4. 승인 문서 30개를 먼저 색인 대상으로 고정한다.
