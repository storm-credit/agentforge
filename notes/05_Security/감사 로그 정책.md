# 감사 로그 정책

## 목적

감사 로그는 Agent Forge에서 사용자 요청, 정책 판정, 문서 검색, 모델 호출, 도구 호출, 관리자 변경을 추적하기 위한 보안 기록이다. 장애 분석용 일반 로그와 구분하며, 감사 로그는 무결성, 최소 수집, 검색 가능성, 보존 정책을 동시에 만족해야 한다.

## 원칙

- 모든 실행은 request_id를 가진다.
- 사용자, 에이전트, 정책, 문서, 도구 이벤트는 request_id로 상관관계를 추적할 수 있어야 한다.
- 감사 로그는 append-only로 저장하고 임의 수정/삭제를 금지한다.
- 민감 원문은 기본 저장하지 않고 hash, metadata, summary 중심으로 저장한다.
- 감사 담당자도 필요 범위 내에서만 조회하며, 조회 행위도 감사 로그로 남긴다.
- 폐쇄망 환경에서 로그는 외부 SaaS로 전송하지 않는다.

## 이벤트 카테고리

| 카테고리 | 이벤트 예시 | 필수 여부 | 보존 기준 |
|---|---|---|---|
| Authentication | login.success, login.failure, token.expired | 필수 | 보안 정책에 따름 |
| Authorization | policy.allowed, policy.denied, acl.filtered | 필수 | 감사 기준 기간 |
| Agent Lifecycle | agent.created, agent.updated, agent.published, agent.disabled | 필수 | 에이전트 수명 + 감사 기간 |
| Ingestion | document.ingested, document.rejected, acl.synced, index.updated | 필수 | 문서 보존 기간 연동 |
| Runtime | request.accepted, request.completed, request.failed | 필수 | 감사 기준 기간 |
| Retrieval | retrieval.started, retrieval.completed, citation.selected | 필수 | 원문 없이 metadata 중심 |
| Guardrail | prompt_injection.detected, pii.masked, output.blocked | 필수 | 보안 이벤트 장기 보존 |
| Tool | tool.requested, tool.allowed, tool.denied, tool.completed | MVP 읽기 도구 필수 | Tool 위험 등급별 보존 |
| Admin | policy.changed, model.changed, tool.enabled, role.changed | 필수 | 장기 보존 |
| Audit Access | audit_log.viewed, audit_export.created | 필수 | 장기 보존 |

## 원문 저장 정책

| 데이터 | 기본 저장 | 허용 조건 | 비고 |
|---|---|---|---|
| 사용자 질의 원문 | 저장하지 않음 | 디버그 모드와 사용자/관리자 승인 시 제한 기간 | hash와 길이, 분류 태그 저장 |
| 모델 응답 원문 | 저장하지 않음 | 품질 평가 샘플링 또는 사고 조사 승인 시 | 개인정보 masking 후 저장 |
| 검색 chunk 원문 | 저장하지 않음 | citation id, document_id, chunk_id만 저장 | 원문은 Document Store 권한으로 조회 |
| 도구 입력/출력 원문 | 저장하지 않음 | 읽기 도구의 비민감 metadata만 | 쓰기 도구 확장 시 별도 승인 |
| 관리자 변경 diff | 저장 | secret 값은 masking | 변경 전/후 값은 민감도에 따라 masking |

## 무결성과 보존

- Audit Log Store는 append-only 테이블 또는 WORM 성격의 저장소를 우선한다.
- 로그 batch 단위로 hash chain 또는 서명 값을 남겨 변조 탐지가 가능해야 한다.
- 관리자도 직접 UPDATE/DELETE 권한을 갖지 않는다.
- 보존 기간은 조직 정책에 맞추되, MVP 기본값은 최소 1년을 권장한다.
- 보존 기간 만료 시 삭제 이벤트와 삭제 범위를 별도 감사 로그로 남긴다.
- 보안 이벤트, 관리자 변경, 감사 로그 조회 기록은 일반 실행 로그보다 긴 보존 기간을 적용한다.

## 조회와 반출 통제

| 행위 | 허용 주체 | 통제 |
|---|---|---|
| 본인 실행 이력 조회 | End User | 원문 없는 요약과 상태 중심 |
| 소유 에이전트 실행 로그 조회 | Agent Owner | 소유 에이전트 범위, 민감 필드 masking |
| 전체 감사 로그 조회 | Security Auditor | 사유 입력, 조회 로그 기록 |
| 로그 export | Security Auditor + 승인 | 기간/범위 제한, export hash 기록 |
| 원문 복구 조회 | 별도 승인자 | 사고 조사 ticket, 시간 제한, 조회 기록 |

## 알림 대상 이벤트

다음 이벤트는 보안 알림 또는 운영 알림으로 연결한다.

- ACL 없는 문서 ingestion 시도
- 제한/기밀 문서에 대한 반복 접근 거부
- prompt injection 탐지
- PII masking 발생량 급증
- 미등록 도구 호출 시도
- 관리자 정책 완화 또는 Tool Registry 활성화
- 감사 로그 export
- 서비스 계정의 비정상 접근 패턴

## MVP와 이후 확장

| 항목 | MVP | 이후 확장 |
|---|---|---|
| 로그 저장 | 내부 Audit Log Store, append-only | WORM/object lock, SIEM 연동 |
| 로그 내용 | metadata/hash/summary 중심 | 위험 기반 원문 샘플링 |
| 검색 | request_id, actor, agent, event_type 기준 | 전문 검색, 이상탐지 |
| 무결성 | append-only 권한 분리 | hash chain, 전자서명, 외부 보관 |
| 알림 | 주요 보안 이벤트 rule 기반 | UEBA, risk score |
