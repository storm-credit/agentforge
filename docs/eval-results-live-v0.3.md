## 2. 답변 확신 게이트(`answer_min_score`) — 정직한 음성 결과

`retrieval_min_score`(컨텍스트 진입 0.53)와 분리된 **답변 확신 게이트**를 신설(env `AGENT_FORGE_ANSWER_MIN_SCORE`, 기본 0.0=off): top 점수 < answer_min이면 LLM 호출 없이 거부.

측정된 top_score(결정적):
- 고쳐야 할 과답변: **c07=0.587**
- 죽이면 안 되는 정상답변(약매치): **c06=0.558, c12=0.567, c14=0.554** (그 외 c01~c05,c10,c11,c13,c15 = 0.63~0.75)

answer_min(θ) 스윕 — 각 케이스는 top<θ면 거부(측정 top_score에서 결정적 도출):

| θ | c07 거부?(목표) | 함께 잘못 거부되는 정상답변 | 판정 |
|---|---|---|---|
| 0.00 (off, 현재) | ✗ | 없음 | 현 상태 |
| 0.56 | ✗ (0.587≥0.56) | c06(0.558), c14(0.554) | 최악(c07 못 잡고 정상 2개 손실) |
| 0.59 | ✓ | c06, c12(0.567), c14 | c07 잡지만 정상 3개 손실 |
| 0.62 | ✓ | c06, c12, c14 | 동일 |

**결론(정량 확정): 스칼라 점수 게이트로 c07을 거부하면서 정상답변을 보존하는 θ는 존재하지 않는다.** 과답변 c07(0.587)이 정상답변 c06/c12/c14(0.55~0.57)보다 *더 높은* 벡터 유사도를 받기 때문이다(접근 가능한 보안 지침이 질의와 표면적으로 유사). 즉 **거부 규율은 벡터 점수만으로 못 고친다.**

## 3. 함의 — 리랭킹/LLM-judge가 정공법
- **rerank(cross-encoder)**: 질의-문서 쌍을 직접 채점하므로 c07의 *느슨하게 관련된* 접근가능 문서를 낮게 줄 수 있다(벡터 코사인이 못 하는 구분).
- **LLM-judge faithfulness**: "이 문서가 질문에 실제로 답하는가"를 의미 수준에서 판정.
- 둘 다 사내 cross-encoder/운영 모델 가용 시 우선 후속(Ollama는 rerank 미지원 — 본 슬라이스에서 확인).
