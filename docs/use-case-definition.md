# MVP 유스케이스 정의 v0.1

## 유스케이스

사내 문서 기반 RAG 에이전트 빌더.

관리자는 승인된 사내 문서 묶음, 승인 모델, 답변 정책, 접근 정책을 선택해 Q&A 에이전트를 만든다. 사용자는 업무 질문을 입력하고, 본인 권한으로 접근 가능한 문서에 근거한 답변을 받는다.

## MVP로 선택한 이유

| 기준 | 이유 |
|---|---|
| 업무 수요 | 규정, 매뉴얼, 절차, 지침은 반복 질문이 많다 |
| 보안 검증 | 문서 ACL, 부서 권한, 문서등급 필터를 실제로 테스트할 수 있다 |
| 기술 검증 | 수집, 파싱, 임베딩, 검색, rerank, citation, audit log를 모두 포함한다 |
| 확장 가치 | 동일한 런타임과 감사 구조를 DB, 그룹웨어, ERP 에이전트에 재사용할 수 있다 |
| 일정 현실성 | 쓰기 작업과 복잡한 시스템 연동을 제외하면 8주 MVP 데모가 가능하다 |

## 페르소나

| 페르소나 | 역할 | MVP 성공 기준 |
|---|---|---|
| Agent 관리자 | 파일럿 에이전트를 설정하고 게시한다 | 가이드 기반으로 기본 에이전트 생성 가능 |
| 문서 소유자 | 승인 문서와 메타데이터를 제공한다 | 유효 문서, 폐기 문서, 제한 문서를 구분 |
| 업무 사용자 | 질문하고 citation을 확인한다 | 문서 검색 시간을 줄이고 답변 근거를 확인 |
| 보안/감사 담당자 | 권한 동작과 로그를 검토한다 | 권한 없는 문서 노출 0건 확인 |
| 플랫폼 운영자 | 색인, 실행, 오류를 모니터링한다 | 실패 원인과 재처리 대상을 식별 |

## 파일럿 후보

| 후보 | 설명 | MVP 적합도 | 비고 |
|---|---|---|---|
| 보안/개인정보 규정 Q&A | 데이터 처리, 보안 기준, 외부 반출 기준 질의 | 높음 | ACL과 audit log 검증에 가장 적합 |
| 인사/총무 업무 매뉴얼 Q&A | 휴가, 출장, 복리후생, 비용 처리 기준 질의 | 높음 | 반복 문의 감소 효과가 명확 |
| IT 운영 절차 Q&A | 장애, 계정, 사내 시스템 이용 절차 질의 | 중간 | 문서 최신성 관리가 중요 |

권장 1차 파일럿은 보안/개인정보 규정 Q&A 또는 인사/총무 업무 매뉴얼 Q&A이다. 최종 선택은 1주차에 문서 확보 가능성과 담당 부서 참여 의지로 결정한다.

## End-to-End 흐름

| 단계 | 행위자 | 행위 | 시스템 동작 | 산출/로그 |
|---|---|---|---|---|
| 1 | 문서 소유자 | 파일럿 문서 후보와 접근 등급 제공 | 수집 메타데이터 준비 | 문서 목록 |
| 2 | 관리자 | 문서 묶음 등록 | 파싱, 청킹, 임베딩, 색인 | 색인 상태 로그 |
| 3 | 관리자 | 목적, 모델, 답변 정책, 접근 범위 설정 | Agent Card와 정책 버전 저장 | Agent v1 |
| 4 | 보안 담당자 | 허용/차단 계정으로 테스트 | ACL 기반 검색 적용 | 보안 테스트 결과 |
| 5 | 사용자 | 업무 질문 입력 | 허용 문서만 검색 | 질문/검색 로그 |
| 6 | 시스템 | 근거 기반 답변 생성 | citation 포함 답변 반환 | 답변/citation 로그 |
| 7 | 사용자 | 피드백 입력 | 유용성/오류 피드백 저장 | 피드백 로그 |
| 8 | 운영자 | 저품질/실패 케이스 확인 | 재색인 또는 정책 변경 대상 식별 | 개선 백로그 |

## 기능 요구사항

### Agent Builder

- 에이전트 이름, 설명, 소유 부서, 담당자를 입력할 수 있어야 한다.
- 문서 묶음, 모델, 시스템 프롬프트, 답변 정책을 선택할 수 있어야 한다.
- 부서, 사용자 그룹, 문서등급 기준으로 공개 범위를 설정할 수 있어야 한다.
- 에이전트 상태를 draft, review, published, archived로 관리할 수 있어야 한다.
- 버전 이력과 변경 사유가 남아야 한다.

### 문서 수집

- MVP에서는 PDF, DOCX, XLSX, TXT, Markdown을 지원한다.
- 문서명, 소유 부서, 버전, 등급, 접근 그룹, 등록일을 저장한다.
- 색인 상태와 실패 사유를 확인할 수 있어야 한다.
- 문서 갱신 시 재색인 필요 상태를 표시해야 한다.
- 전사 자동 동기화보다 파일럿 문서 묶음의 정확한 처리를 우선한다.

### 검색과 답변

- 답변 생성 전 사용자, 부서, 그룹, 문서등급으로 검색 후보를 제한한다.
- 답변에는 문서명과 페이지, 섹션, 문단 ID, 또는 원문 위치가 포함되어야 한다.
- 근거가 부족하면 추측하지 않고 한계를 안내해야 한다.
- 문서 기준이 충돌하면 충돌 가능성을 알리고 근거를 나란히 제시한다.
- 질문, 검색 문서, 답변, 모델, 정책 버전을 추적할 수 있어야 한다.

### 운영과 감사

- 관리자 화면에서 문서 색인 상태, 에이전트 상태, 최근 실행 로그를 볼 수 있어야 한다.
- audit log에는 사용자, 시각, 질문, 검색 문서, 답변 상태, 모델, 정책 버전, 오류 여부가 남아야 한다.
- 민감 원문을 저장하기 전에 마스킹과 보관 기준을 정해야 한다.
- 파일럿 기간의 실패 질문과 낮은 평가는 개선 백로그로 전환한다.

## 비기능 요구사항

| 영역 | 요구사항 |
|---|---|
| 보안 | 권한 없는 문서는 검색 후보에 포함되지 않아야 한다 |
| 신뢰성 | 색인 실패, 모델 오류, 검색 결과 없음, 권한 차단 상태가 구분되어야 한다 |
| 성능 | 파일럿 문서와 사용자 규모에서 업무 사용 가능한 응답 시간을 제공해야 한다 |
| 감사 | 주요 실행 단계와 정책 버전을 추적할 수 있어야 한다 |
| 운영성 | 실패 문서, 실패 실행, 재색인 대상을 운영자가 식별할 수 있어야 한다 |
| 사용성 | 관리자가 코드 변경 없이 기본 파일럿 에이전트를 만들 수 있어야 한다 |

## 수용 기준

| ID | 기준 | 검증 방법 |
|---|---|---|
| AC-001 | 관리자가 문서 묶음과 답변 정책을 설정해 에이전트를 생성할 수 있다 | 관리자 시나리오 테스트 |
| AC-002 | 사용자는 권한이 있는 문서에서만 검색 결과를 받는다 | 권한이 다른 테스트 계정으로 검증 |
| AC-003 | 권한 없는 문서는 citation, 답변, 로그 조회 화면에 노출되지 않는다 | 보안 테스트 |
| AC-004 | 답변에는 문서명과 위치 정보가 포함된 citation이 표시된다 | 평가 질문 세트 |
| AC-005 | 근거 부족 질문에는 한계 안내가 제공된다 | 부정 질문 테스트 |
| AC-006 | 실행 로그에서 질문, 검색 문서, 답변 상태, 정책 버전을 확인할 수 있다 | 운영자 점검 |
| AC-007 | 색인 실패와 실패 사유가 관리자에게 표시된다 | 실패 파일 수집 테스트 |
| AC-008 | 파일럿 평가 질문 기준 유용 답변 비율이 80% 이상이다 | QA/Eval 리포트 |

## 평가 데이터

- 파일럿 문서 30~100개 내외로 시작한다.
- 사실 확인형, 절차 안내형, 조건 비교형, 권한 차단형, 근거 부족형 질문을 포함한다.
- 각 질문에는 기대 답변, 허용 문서, 금지 문서, citation 기준을 함께 정의한다.
- 결과는 정답성, citation 적합성, 권한 준수, 답변 명확성으로 평가한다.

## 제외 시나리오

- 메일 발송, 결재 생성, 업무 프로세스 실행
- ERP/그룹웨어 쓰기 변경
- 관리자 승인 없는 전사 문서 자동 수집
- 사용자가 볼 수 없는 문서의 존재를 암시하는 답변
- 근거 없는 일반 지식을 사내 정책처럼 제시하는 답변
