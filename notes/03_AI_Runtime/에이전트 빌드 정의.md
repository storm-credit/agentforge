# 에이전트 빌드 정의

## 정의

에이전트 빌드는 특정 업무 목적을 수행하도록 모델, 지식, 도구, 권한, 승인정책, 실행흐름, 로그정책을 하나로 묶어 배포 가능한 상태로 만든 실행 패키지다.

## 구성 요소

```yaml
agent_id: security_policy_assistant
name: 보안규정 상담 에이전트
purpose: 사내 보안 규정과 절차를 근거 기반으로 안내한다
model: local-llm
knowledge_sources:
  - security_policy_docs
tools:
  - document_search
permissions:
  document_acl: enabled
  allowed_departments:
    - all_employees
approval_required:
  - confidential_export
workflow:
  - classify_question
  - retrieve_allowed_documents
  - generate_answer_with_citation
  - security_check
  - log_run
quality_gates:
  - citation_required
  - acl_filter_required
  - pii_masking
logs:
  - user_id
  - prompt_hash
  - retrieved_docs
  - answer_hash
  - risk_flags
```

## 빌더가 만드는 것

Agent Builder는 에이전트 자체의 코드만 만드는 것이 아니라, 실행 가능한 설정과 정책 묶음을 만든다.
