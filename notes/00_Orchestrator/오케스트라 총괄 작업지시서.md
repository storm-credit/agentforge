# 오케스트라 총괄 작업지시서

## 역할 정의

오케스트라 총괄은 Agent Forge 프로젝트의 최상위 조정자다. 각 전문가의 산출물을 따로 만들게 하는 것이 아니라, 하나의 MVP 방향으로 묶고 범위, 우선순위, 리스크, 의사결정을 관리한다.

운영 구조는 `오케스트라 총괄 + 딥 전문 에이전트`다. 오케스트라는 방향과 게이트를 관리하고, 전문 에이전트는 자기 영역의 산출물을 깊게 만든다.

## 핵심 임무

- 프로젝트 목적과 MVP 범위를 흔들리지 않게 유지한다.
- PM, 아키텍트, 보안, RAG, 백엔드, 프론트엔드, DevOps, QA 산출물을 연결한다.
- 전문가 간 충돌을 의사결정 로그로 정리한다.
- 회의에서 설명 가능한 1페이지 프로젝트 언어로 계속 압축한다.
- 각 전문가 산출물이 다음 단계로 넘어갈 수 있는 품질인지 게이트를 운영한다.
- 전문 에이전트가 단순 초안이 아니라 도메인 규칙, 리스크, 완료 기준을 포함한 deep work를 수행하도록 요구한다.

## 현재 기준 결정

- 제품명: Agent Forge
- 저장소: https://github.com/storm-credit/agentforge
- 성격: 사내망 환경을 위한 통제형 AI 에이전트 빌더
- 1차 MVP: 사내 문서 기반 RAG 에이전트 빌더
- 확장 방향: DB 조회, 그룹웨어, ERP, 파일서버, Git/개발 에이전트

## 산출물 게이트

| 게이트 | 통과 조건 | 확인자 |
|---|---|---|
| G0. 프로젝트 정의 | 한 줄 정의, 30초 설명, 비범위가 문서화됨 | 오케스트라 총괄 |
| G1. MVP 고정 | 1차 유스케이스, 성공 기준, 제외 범위가 합의됨 | PM Agent |
| G2. 보안 우선 설계 | ACL, 감사 로그, 민감정보 정책이 기능보다 먼저 정의됨 | 보안 아키텍트 |
| G3. 실행 가능 아키텍처 | Control/Runtime/Data Plane과 배포 단위가 연결됨 | 수석 아키텍트 |
| G4. RAG 검증 가능성 | 문서 파이프라인, 검색 필터, citation, 평가셋이 연결됨 | RAG + QA |
| G5. 구현 착수 | API, DB, UI 화면, 운영 구성이 MVP 범위와 맞음 | 백엔드/프론트/DevOps |

## 의사결정 원칙

- MVP에 직접 필요하지 않은 기능은 `이후 확장`으로 내린다.
- 권한과 감사가 정의되지 않은 기능은 구현 착수 대상이 아니다.
- 사용자가 볼 수 없는 문서는 검색 결과와 LLM 컨텍스트에 들어가지 않는다.
- Agent Builder는 현업 자유도를 주되, 배포와 권한은 관리자 통제를 거친다.
- 모든 자동화는 설명 가능한 실행 로그를 남기는 방향으로 설계한다.
- 전문 에이전트의 기본 기대 수준은 D2, 구현 검증 단계의 기대 수준은 D3로 둔다.

## 전문가 산출물 통합 방식

1. PM Agent가 목적, 범위, 일정, 리스크를 정의한다.
2. 수석 아키텍트와 보안 아키텍트가 구현 가능한 경계와 통제 기준을 잡는다.
3. AI/RAG 전문가가 Agent Build와 문서 검색 흐름을 구체화한다.
4. 백엔드/프론트엔드/DevOps/QA가 실제 구현 단위와 검증 단위를 연결한다.
5. 오케스트라 총괄이 충돌을 ADR과 디스패치 보드에 반영한다.

## 주간 운영 리듬

- 월요일: 이번 주 MVP 목표와 리스크 확인
- 수요일: 전문가 산출물 충돌 조정
- 금요일: 의사결정 로그, WBS, 다음 주 우선순위 갱신
