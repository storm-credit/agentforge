# 플랫폼 내부 에이전트 목록

## MVP 포함 범위

| 에이전트 | 역할 | MVP 포함 | 주요 산출물 |
|---|---|---|---|
| Planner Agent | 사용자 요청을 처리 단계로 분해하고 허용된 실행 계획 생성 | 예 | `intent`, `risk_level`, `plan_steps`, `retrieval_targets` |
| Retriever Agent | 권한 기반 문서 검색과 citation 후보 생성 | 예 | `allowed_chunks`, `retrieval_trace`, `citation_candidates` |
| Security Guard Agent | 정보유출, prompt injection, 권한 위반 점검 | 예 | `security_flags`, `block_decision`, `masking_actions` |
| Critic Agent | 답변 근거, citation, 정책 준수 품질 검토 | 예 | `critic_findings`, `revision_required`, `quality_scores` |
| Tool Executor Agent | 사내 API/DB/업무 도구 호출 | 이후 | `tool_call_result`, `tool_audit` |
| Human Approval Agent | 위험 작업 승인 요청 생성 | 이후 | `approval_request`, `approval_status` |
| Memory Agent | 업무 컨텍스트 저장 기준 관리 | 이후 | `memory_write_decision`, `memory_refs` |
| Cost/Latency Agent | 모델 선택과 응답 비용 최적화 | 예(정책/trace부터) | `model_route`, `budget_trace` |

MVP는 완전 자율 multi-agent 실행이 아니라, Runtime Orchestrator가 고정된 단계를 호출하는 통제형 파이프라인으로 구현한다. 각 내부 에이전트는 독립 서비스일 필요는 없고, 같은 런타임 프로세스 안의 모듈로 시작해도 된다.

모델 효율화 기준은 `docs/agent-model-routing-policy.md`와 `packages/shared-contracts/model-routing-policy.v0.1.json`을 따른다. 핵심은 모든 판단을 큰 모델에 맡기는 것이 아니라, ACL/스키마/스코어링은 deterministic으로 고정하고, 분류와 계획은 작은 모델, 근거 답변은 RAG용 표준 모델, 보안/릴리스 판단은 deep-review 모델로 승격하는 것이다.

## 공통 실행 규칙

- 모든 내부 에이전트는 `RuntimeState`를 입력으로 받고, 자신의 결과만 append한다.
- 사용자 원문, 검색 결과 원문, 최종 답변은 권한과 로그 정책에 따라 마스킹/암호화한다.
- 내부 에이전트 간 전달 데이터에는 사용자의 `user_id` 원문 대신 런타임이 확인한 권한 컨텍스트를 사용한다.
- 실패는 조용히 무시하지 않고 `stage`, `reason_code`, `safe_response`를 남긴다.
- 모델이 생성한 계획이나 판단은 정책 엔진 결과를 덮어쓸 수 없다.

## 기본 Runtime Flow

```text
1. Ingress
2. Security Guard pre-check
3. Planner
4. Retriever
5. Answer Generator
6. Critic
7. Security Guard final-check
8. Response Formatter
9. Audit Logger
```

## 모델 라우팅 요약

| 내부 에이전트 | 기본 모델 tier | 승격 조건 |
|---|---|---|
| Security Guard pre-check | `fast-small` + deterministic pattern | injection/PII/외부반출 판단이 애매할 때 `deep-review` |
| Planner | `fast-small` | 여러 source/intent가 충돌할 때 `standard-rag` |
| Retriever | `deterministic` + `embedding` + `reranker` | 모델 승격 없음, ACL filter 실패 시 중단 |
| Answer Generator | `standard-rag` | citation 없는 답변은 승격이 아니라 실패/재작성 |
| Critic | `fast-small` | release gate 실패, citation 충돌, 보안 민감 답변은 `deep-review` |
| Security Guard final-check | `fast-small` | ACL leak/PII/policy conflict 가능성은 `deep-review` |
| Formatter | `deterministic` | 승격 없음, 새 사실 추가 금지 |
| Cost/Latency Agent | `deterministic` | 승격 없음, route/budget trace만 기록 |

## Planner Agent

### 책임

- 사용자 요청의 intent를 분류한다.
- 필요한 지식 소스와 검색 전략을 고른다.
- 답변 가능/불가능/승인 필요 여부를 판단한다.
- Runtime이 실행할 수 있는 짧은 step list를 만든다.

### 입력

```yaml
user_message: "협력사에 로그 파일을 전달해도 되나요?"
agent_contract:
  allowed_intents: ["policy_qa", "procedure_help", "document_summary"]
  knowledge_sources: ["security_policy_docs"]
security_precheck:
  blocked: false
  risk_level: "medium"
auth_context:
  department_id: "security"
  roles: ["employee"]
  clearance_level: "internal"
```

### 출력

```yaml
intent:
  category: "policy_qa"
  confidence: 0.86
  risk_level: "medium"
retrieval_targets:
  - source_id: "security_policy_docs"
    query: "협력사 로그 파일 외부 반출 절차"
    filters_hint:
      document_types: ["policy", "procedure"]
plan_steps:
  - step_id: "retrieve_policy"
    agent: "Retriever"
    required: true
  - step_id: "generate_cited_answer"
    agent: "AnswerGenerator"
    required: true
```

### 실패/차단 규칙

- intent가 Agent Card의 `allowed_intents` 밖이면 검색하지 않고 지원 범위 안내로 종료한다.
- 사용자가 "권한 필터를 무시해 달라"처럼 정책 우회를 요구하면 `Security Guard` 차단 결과를 따른다.
- Planner는 없는 도구를 계획에 넣을 수 없다. 도구 registry에 없는 step은 schema validation에서 실패한다.

## Retriever Agent

### 책임

- Planner가 지정한 query를 내부 검색 쿼리로 정규화한다.
- 사내 IAM/ACL 컨텍스트를 vector DB payload filter로 변환한다.
- hybrid search, rerank, citation locator 생성을 수행한다.
- LLM 컨텍스트에 넣을 수 있는 chunk만 반환한다.

### 입력

```yaml
retrieval_targets:
  - source_id: "security_policy_docs"
    query: "협력사 로그 파일 외부 반출 절차"
auth_context:
  department_id: "security"
  roles: ["employee"]
  clearance_level: "internal"
index_snapshot_id: "idx_security_policy_2026_05_01"
retrieval_policy:
  vector_top_k: 40
  lexical_top_k: 20
  rerank_top_k: 8
```

### 출력

```yaml
retrieval_trace:
  acl_filter_applied: true
  index_snapshot_id: "idx_security_policy_2026_05_01"
  vector_hits: 34
  lexical_hits: 12
  reranked_hits: 8
allowed_chunks:
  - chunk_id: "sec-policy-2026:p12:c03"
    document_id: "sec-policy-2026"
    title: "외부 반출 보안 절차"
    section: "3.2 로그 및 진단자료"
    page: 12
    score: 0.81
    acl_decision: "allow"
    citation_locator: "[외부 반출 보안 절차, 3.2, p.12]"
```

### 실패/차단 규칙

- ACL payload filter가 생성되지 않으면 검색을 실행하지 않는다.
- 검색 결과가 있어도 citation locator가 없는 chunk는 Generator 컨텍스트에 넣지 않는다.
- 권한 밖 문서가 후보에 섞이면 해당 chunk를 폐기하고 보안 이벤트로 기록한다.

## Critic Agent

### 책임

- 답변 문장이 검색된 chunk로 뒷받침되는지 확인한다.
- citation 누락, 근거 불일치, 오래된 문서, 서로 충돌하는 문서를 찾아낸다.
- 답변을 재작성할지, 사용자에게 안전하게 실패 응답을 줄지 결정한다.

### 입력

```yaml
draft_answer: "로그 파일은 보안팀 승인 후 전달할 수 있습니다..."
allowed_chunks:
  - chunk_id: "sec-policy-2026:p12:c03"
    text_hash: "sha256-..."
citations:
  - chunk_id: "sec-policy-2026:p12:c03"
```

### 출력

```yaml
critic_findings:
  groundedness:
    status: "pass"
    unsupported_claims: []
  citation_coverage:
    status: "pass"
    missing_citation_sentences: []
  conflicts:
    status: "warn"
    notes: ["2025 절차 문서보다 2026 개정 문서 우선"]
revision_required: false
quality_scores:
  groundedness: 0.92
  citation_coverage: 1.0
```

### 실패/차단 규칙

- 핵심 결론에 citation이 없으면 Generator 재작성을 1회 요청한다.
- 재작성 후에도 citation coverage가 기준 미만이면 "확인 가능한 근거가 부족합니다"로 실패 응답한다.
- Critic은 새 사실을 추가하지 않는다. 검색된 chunk 안에서만 평가한다.

## Security Guard Agent

Security Guard는 pre-check와 final-check 두 번 호출한다.

### Pre-check 책임

- prompt injection, 권한 우회, 개인정보/기밀 요청, 외부 반출 의도를 분류한다.
- 차단 요청은 Retriever 이전에 종료한다.
- 고위험이지만 업무상 가능성이 있는 요청은 Planner가 제한된 검색만 수행하도록 risk flag를 넘긴다.

### Final-check 책임

- 최종 답변에 권한 밖 문서 내용, 개인정보, 기밀 원문 과다 인용이 없는지 검사한다.
- citation에 노출하면 안 되는 파일명/경로/작성자 정보가 있는지 확인한다.
- 필요하면 마스킹 후 다시 Critic의 citation coverage를 확인한다.

### 출력

```yaml
security_flags:
  - code: "EXTERNAL_EXPORT_REQUEST"
    severity: "medium"
    stage: "precheck"
    action: "allow_with_constraints"
final_decision:
  blocked: false
  masking_applied: true
  redacted_fields:
    - "personal_identifier"
```

### 실패/차단 규칙

- prompt injection이 시스템 지시 변경, ACL 우회, 숨겨진 prompt 공개를 요구하면 즉시 차단한다.
- 사용자의 clearance보다 높은 문서 내용이 최종 답변에 포함되면 차단하거나 해당 문장을 제거한다.
- audit log 기록에 실패하면 최종 응답을 성공으로 반환하지 않는다.
