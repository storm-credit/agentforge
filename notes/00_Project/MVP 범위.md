## MVP 이름

사내 문서 기반 RAG 에이전트 빌더

## MVP 정의

관리자가 승인된 사내 문서 묶음, 모델, 답변 정책, 접근 권한을 설정해 RAG 기반 업무 Q&A 에이전트를 만들고, 사용자는 본인 권한으로 접근 가능한 문서에서만 citation이 포함된 답변을 받는다.

## 우선 검증 가설

| 가설 | 검증 방법 |
|---|---|
| 사내 문서 Q&A는 반복 문의를 줄일 수 있다 | 파일럿 부서 질문 세트와 사용자 피드백 |
| 권한 필터를 검색 전에 적용하면 문서 노출 리스크를 통제할 수 있다 | 권한별 테스트 계정과 금지 문서 질의 |
| citation과 audit log가 있어야 업무 활용 승인을 받을 수 있다 | 보안/감사 리뷰와 데모 시나리오 |
| 관리자가 직접 에이전트를 설정할 수 있어야 확장성이 생긴다 | Agent Studio 설정 시나리오 테스트 |

## 포함 범위

- 문서 업로드 또는 수집 등록
- PDF, DOCX, XLSX, TXT, Markdown 기본 파싱
- chunking, embedding, vector index 생성
- 사용자/부서/문서등급 기반 검색 필터
- 근거 citation이 포함된 답변
- 근거 부족 시 한계 안내 답변
- 에이전트 카드 등록과 버전 관리
- 에이전트 draft/review/published 상태 관리
- 실행 로그, 검색 문서, 답변 로그 저장
- 관리자용 최소 Agent Studio 화면
- 테스트 콘솔과 파일럿 피드백 수집

## 제외 범위

- ERP/그룹웨어 쓰기 작업
- 메일 자동 발송
- 결재 자동 처리
- 전사 모든 문서 연결
- 완전 자율 멀티 에이전트
- 고급 비용 최적화와 자동 모델 라우팅
- 문서 보안 등급 자동 판정의 완전 자동화
- 장기 보관 정책과 법무 검토가 필요한 전사 감사 저장소

## MVP 업무 흐름

| 단계 | 설명 | 산출물 |
|---|---|---|
| 문서 후보 선정 | 파일럿 부서가 공개 가능 문서와 등급을 정리 | 문서 목록, 접근 그룹 |
| 문서 등록/색인 | 문서 파싱, 청킹, 임베딩, 색인 상태 확인 | vector index, 색인 로그 |
| 에이전트 설정 | 이름, 목적, 모델, 문서 묶음, 권한, 답변 정책 설정 | Agent Card v1 |
| 권한 검증 | 테스트 계정으로 허용/차단 케이스 확인 | ACL 테스트 결과 |
| 사용자 질의 | 파일럿 사용자가 업무 질문 입력 | 질문/검색/답변 로그 |
| 평가/개선 | 답변 유용성, citation, 실패 케이스 분석 | 평가 리포트, v0.2 백로그 |

## MVP 성공 기준

- 권한 없는 문서 노출 0건
- 답변의 근거 citation 포함률 95% 이상
- 샘플 질문 기준 정답/유용 답변 비율 80% 이상
- 주요 실행 단계가 audit log에 남을 것
- 관리자 1명이 새 문서 에이전트를 설정할 수 있을 것

## MVP 완료 조건

- [ ] 파일럿 문서 묶음 1개 이상 등록
- [ ] Agent Studio에서 에이전트 생성/수정/게시 가능
- [ ] 권한별 테스트 계정으로 허용/차단 질의 통과
- [ ] citation 포함 답변과 근거 부족 답변 처리 확인
- [ ] 실행 로그에서 질문, 검색 문서, 답변 상태, 정책 버전 확인
- [ ] 평가 질문 세트 결과와 파일럿 개선 백로그 작성
